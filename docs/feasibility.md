# WattWay Feasibility Check

> 핵심 외부 의존성(경로 API, 충전소 데이터)이 실제로 존재하는지 검증한 기술 타당성 분석.
>
> 작성일: 2026-06-18 · 상태: 통과(Pass), 조건부

---

## 1. 검증 동기

"네이버맵·T맵 수준의 경로 추천을 직접 구현하는 것은 비현실적"이라는 전제에서 출발한다.
따라서 설계를 확정하기 전에 두 가지를 검증한다.

1. 자동차 경로를 **기존 서비스 API로** 얼마나 충당할 수 있는가?
2. 고속도로 휴게소 충전소의 **사양·대수·실시간 상태**를 주는 API가 존재하는가?

**결론: 둘 다 가능. 우리가 직접 만들 것은 도로 라우팅 엔진이 아니라 "충전 정차 선택 로직"이다.**

---

## 2. Q1 — 자동차 경로 API

도로 경로 자체는 상용 API로 완전히 해결된다. 직접 라우팅 엔진(OSRM/Valhalla) 운영 불필요.

| 제공사 | 자동차 경로 | 경유지 | 반환 정보 | EV 특화 |
|--------|:---:|:---:|---|:---:|
| 네이버 Directions 5/15 | ✅ | 5 / **15** | 거리, 시간, **통행료**, 유류비, 경로 좌표, 실시간 교통 | ❌ |
| 카카오모빌리티 길찾기 | ✅ | 다중 | 거리·시간·대안경로·실시간 교통 | ❌ |
| TMAP | ✅ | **다중 + 경유지 최적화 + 경로 매트릭스** | 거리·시간·교통 | ⚠️ |

### 핵심 발견

> 네이버·카카오·TMAP **누구도 "배터리 잔량 기반으로 충전소를 자동 경유지에 삽입하는"
> EV 전용 경로 API를 공개하지 않는다.** TMAP 앱의 EV 충전 경로는 내부 기능이며,
> 외부 공개 API는 "일반 자동차 경로 + 충전소 검색"을 분리 제공하는 수준이다.

### 함의

- **충전 정차 최적화 로직은 우리가 직접 구현**한다. 단, 이는 도로망 그래프 탐색이 아니라
  **외부 경로 API를 도구로 호출하며 후보 충전소를 평가·선택하는 오케스트레이션**이다.
- TMAP **경로 매트릭스(Route Matrix)** API가 "후보 충전소 간 거리·시간 일괄 계산"에 적합 →
  정차 최적화의 핵심 도구.
- 따라서 자체 도로 엔진(OSRM/Valhalla) 운영은 불필요. → **ADR-0003 수정**.

## 3. Q2 — 휴게소 충전소 사양·대수·실시간 상태 API

### ① 한국환경공단_전기자동차 충전소 정보 (정답)

- 출처: 공공데이터포털 `data.go.kr/data/15076352`
- 엔드포인트: `http://apis.data.go.kr/B552584/EvCharger`
  - `getChargerInfo` — 충전기 **정보**: 커넥터 타입(DC차데모·AC완속·DC콤보·AC3상 등),
    **충전용량(kW)**, 위경도, 주소, 운영기관, 충전소 구분
  - `getChargerStatus` — **실시간 상태**: `충전대기 / 충전중 / 운영중지 / 점검중 / 통신장애`
    + 상태갱신일시
- 범위: 전국 약 7만소 / 30만기 — **고속도로 휴게소 충전소 포함**.
  휴게소를 위치/명칭/운영기관으로 필터링하면 **휴게소별 대수·사양·실시간 상태**를 모두 획득.
- 갱신: 운영사 푸시 기반 **준실시간**(지연·신뢰도 편차 존재 → 안내용으로 충분, 초단위 보장 아님).

### ② 한국도로공사_휴게소 전기차·수소차 충전소 현황 (보조)

- 출처: `data.go.kr/data/15085543`, `data.ex.co.kr`
- 항목: 시설명 / 위치 / 전기차 충전소 유무 / 수소차 유무 — **사양·대수·실시간 없음**.
- 용도: 휴게소 목록·존재 여부 보조용. 사양·대수·실시간은 ①로 채운다.

### ③ 보조 데이터

- 한국전력공사 충전소 운영정보(`data.go.kr/data/3068728`): KEPCO 운영분 (①로 대부분 커버).

## 4. 종합 판정

| 항목 | 가능 | 방법 |
|------|:---:|------|
| 자동차 도로 경로 | ✅ | 네이버/카카오/TMAP 경로 API (직접 구현 X) |
| 통행료·소요시간 | ✅ | 경로 API 반환값 |
| 충전소 사양·대수 | ✅ | 환경부 `getChargerInfo` |
| 휴게소 충전소 실시간 상태 | ✅(준실시간) | 환경부 `getChargerStatus` |
| 충전 정차 최적화(차별점) | ✅ 직접 구현 | 경로 매트릭스 + 충전소 데이터 자체 로직 |

**한 줄 요약**: 길찾기는 API로 가져오고, 우리는 "잔량 기반으로 어느 휴게소에서 몇 %
충전할지 고르는 두뇌"만 만든다. 인프라가 모두 공개 API로 존재 → **feasibility 통과**.

## 5. 미해소 리스크 (후속 조사 필요)

1. **경로 API 약관·쿼터·과금**: 네이버 NCP / 카카오모빌리티 / TMAP 각각 상업적 이용 정책,
   호출 한도, 과금 체계가 상이. 어느 API를 1차로 채택할지 결정 전 비교 필요.
2. **충전소 상태 신뢰도/지연**: "도착했더니 점검중"에 대비한 **대체 충전소 fallback** 설계 필요.
3. **공공데이터 쿼터**: 환경부 API 개발계정 1,000건/일 제한 → 운영계정 전환 및 **자체 DB 캐싱**
   (Supabase에 동기화 후 서빙)으로 호출량 관리.
