# ADR-0002: 백엔드 플랫폼으로 Supabase 채택

- 상태: Accepted
- 날짜: 2026-06-18

## 맥락

인증, 관계형 데이터, 지리공간 질의(충전소 반경 검색), 실시간 가용성 구독,
파일 저장이 필요하다. MVP를 빠르게 내고 운영 부담을 줄여야 한다.

## 결정

Supabase를 기본 백엔드로 채택한다.

- **Postgres 15 + PostGIS**: 충전소 위치(`geography(Point)`)와 반경/회랑 질의.
- **Auth (JWT)** + **RLS**: 행 수준 접근 제어를 DB에서 일관 적용.
- **Realtime**: 충전소 가용성 변경 구독.
- **Storage**: 차량/리뷰 이미지.
- **Edge Functions**: 충전소 데이터 동기화(Cron) 등 경량 서버 로직.

## 결과

- 장점: 빠른 구축, 통합된 Auth/Geo/Realtime, 운영 단순화.
- 단점: CPU 집약 경로 탐색에는 부적합 → ADR-0003에서 별도 서비스로 분리.
- 위험: 벤더 종속. Postgres 표준에 머무르고 비표준 기능 사용은 ADR로 명시.
