# ADR-0003: 경로 탐색을 전용 서비스로 분리

- 상태: Accepted
- 날짜: 2026-06-18

## 맥락

잔량 기반 경로 + 충전 정차 최적화는 CPU 집약적이고(그래프 탐색, 다수 충전소 후보 평가),
외부 라우팅 엔진(OSRM/Valhalla/Mapbox) 호출과 조합된다. Supabase Edge Function의
실행 시간/리소스 제약에 부적합하다.

## 결정

`apps/routing-service`를 별도 서비스로 둔다(Node.js/TypeScript로 시작).

- 도로망 라우팅은 외부 엔진에 위임(adapter로 추상화, 교체 가능).
- 정차 최적화 알고리즘은 `packages/core`의 순수 함수로 두고 서비스는 I/O만 담당.
- BFF가 이 서비스를 호출하고 결과를 조합한다.

## 결과

- 장점: 독립 확장/배포, 단위 테스트 용이(순수 코어), 엔진 교체 가능.
- 단점: 운영 컴포넌트 추가, BFF→서비스 네트워크 홉.
- 후속: 성능 병목 시 코어 알고리즘을 Go로 이관(I/O 경계가 분리되어 있어 가능).
